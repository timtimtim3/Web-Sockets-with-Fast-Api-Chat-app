# Install ingress-nginx in kind
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml

# Wait for it:
kubectl wait --namespace ingress-nginx \
  --for=condition=ready pod \
  --selector=app.kubernetes.io/component=controller \
  --timeout=180s

# Check:
kubectl get pods -n ingress-nginx

# Port forward ingress:
kubectl port-forward -n ingress-nginx svc/ingress-nginx-controller 8080:80

######
Big Picture: What this Ingress rule does

This config tells the Ingress controller (NGINX):

When a request comes to chatapp.local, route traffic to different services depending on the URL path.

So the traffic flow becomes:

Browser
   ↓
chatapp.local
   ↓
Ingress Controller (nginx)
   ↓
Service (backend)
   ↓
Backend Pods (3 replicas)

Ingress is basically the Kubernetes equivalent of your old NGINX reverse proxy.

# You can see the routing graphically:
kubectl get ingress -n chatapp
kubectl describe ingress chatapp-ingress -n chatapp

# We have:
/auth/*

/messages/*
/ws
